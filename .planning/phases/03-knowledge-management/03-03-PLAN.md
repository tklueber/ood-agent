  <behavior>
    - First `update_markdown()` run with no manifest bootstraps by indexing all non-empty Markdown documents and writing the manifest.
    - Later `update_markdown()` compares current `content_hash` values with manifest entries and sends only `new_paths + changed_paths` to LightRAG `ainsert`.
    - Unchanged files are listed in `diff.unchanged_paths` and are not sent to LightRAG.
    - Deleted files are listed in `diff.deleted_paths` and in the JSON-facing stale/deleted diagnostics, but no deletion call into LightRAG is attempted.
    - If nothing changed, status is `no_changes`, `indexed_documents == 0`, and the command still succeeds.
    - Incompatible manifest `schema_version` raises `ValueError("Manifest schema version is incompatible. Run `ood reindex` to rebuild the knowledge manifest.")`.
  </behavior>
  <action>
    Add `RagEngine.update_markdown(self) -> UpdateResult` as a synchronous wrapper over an async `_aupdate_markdown()` method.

    Implement manifest IO helpers:
    - `_read_manifest(self) -> dict[str, Any] | None` returning `None` when `ood-manifest.json` does not exist.
    - `_manifest_entries_by_path(self, manifest: dict[str, Any]) -> dict[str, dict[str, Any]]`.
    - `_validate_manifest_schema(self, manifest: dict[str, Any]) -> None` raising the exact incompatible-schema `ValueError` message above when `schema_version != 1`.
    - `_build_manifest_diff(parsed_documents, previous_entries) -> ManifestDiff`.

    Update algorithm required by D-01 through D-04 and D-12:
    1. Ensure `settings.storage_dir.mkdir(parents=True, exist_ok=True)`.
    2. Load current parsed Markdown documents, skipping empty body text exactly like index/reindex.
    3. If no manifest exists, treat all non-empty documents as `new_paths` (bootstrap per D-04).
    4. If manifest exists, compare current `content_hash` with previous manifest entry by relative path to compute `new_paths`, `changed_paths`, and `unchanged_paths`.
    5. Compute `deleted_paths` from previous manifest paths missing from current documents and report them only; do not mutate LightRAG internals per D-02.
    6. Detect duplicates by comparing touched documents (`new_paths + changed_paths`) against the full current parsed corpus so new duplicates against unchanged docs are reported per D-12.
    7. If touched non-empty documents exist, call LightRAG/fallback insertion with only touched documents; for local fallback JSON, rewrite the fallback file from all current manifest body texts so query smoke behavior remains complete without calling a Cloud LLM.
    8. Write a fresh manifest for all current non-empty documents, including metadata, warnings, excerpts, body hashes, and duplicate group IDs.
    9. Return `UpdateResult` with `status="updated"` when touched documents were indexed, `status="no_changes"` when no new/changed/deleted/skipped differences exist, and `status="stale_entries"` when only deleted paths are reported.

    Keep all warnings non-blocking per D-05. Do not add automatic stale cleanup or LightRAG deletion behavior per D-02.
