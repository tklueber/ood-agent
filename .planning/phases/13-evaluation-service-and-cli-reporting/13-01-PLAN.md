<objective>
Build the eval orchestration service `EvalRunner` that drives `EvaluationDataset` cases through the public `RagEngine.query()` contract, aggregates retrieval and ticket-intelligence metrics, and produces a typed `EvalRunResult` ready for both the JSON report builder (Plan 02) and the CLI human formatter (Plan 03).

Purpose: This is the EVAL-02 black-box contract — every retrieval backend goes through `RagEngine.query()`. It also locks in the **deliberate Cloud-LLM gate divergence (D-06)**: eval activates the LLM purely on credential availability (`has_llm_credentials`), never on `allow_cloud_llm`. Crucially, because the underlying `RagEngine._aquery` gates LLM usage on `Settings.can_use_cloud_llm` (which is `allow_cloud_llm AND has_llm_credentials`), the runner must construct **effective Settings** for the engine in which `allow_cloud_llm` is forced to True whenever credentials are present — otherwise the divergence would be cosmetic only and `meta.llm_used` would never flip to True without `OOD_ALLOW_CLOUD_LLM=true`.

Skip handling (D-08), crash isolation (D-10), and hard-error propagation (D-09 for `IndexMissingError`) live here so reports never lose data and the CLI can apply the correct exit code.

Output:
- `src/ood/eval_runner.py` with `EvalRunner`, `EvalCaseResult`, `EvalRunSummary`, `EvalRunResult`, plus the `_engine_settings` helper that forges effective Settings.
- `Settings.eval_dataset_path` bound to `OOD_EVAL_DATASET` ENV
- `EvaluationCase.expected_llm_answer: str | None` (optional — used by skip logic)
- `tests/test_eval_runner.py` covering happy/skip/error/credential-gate/effective-settings/IndexMissing paths
</objective>
